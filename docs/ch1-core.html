MiniMindConfig — 模型的"控制台"

所有超参数集中在这一个类里。理解每个参数的含义和默认值，是读懂后续所有代码的前提。

hidden_size = 768

每个 token 被表示为一个 768 维向量。整个模型的"宽度"。越大模型越强，但显存和计算也越多。

参考：GPT-2 Small = 768，LLaMA-7B = 4096，GPT-4 = ~12800（估计）

num_hidden_layers = 8

Transformer Block 堆叠的层数，模型的"深度"。每一层都会进一步提炼 token 的语义表示。

参考：GPT-2 Small = 12，LLaMA-7B = 32

vocab_size = 6400

词表大小，tokenizer 能把文字切成的最小单元数。6400 很小（LLaMA 是 32000），适合 demo 和学习。

num_attention_heads = 8

多头注意力的头数。每个头独立学习不同的注意力模式。head_dim = hidden_size / heads = 768/8 = 96

num_key_value_heads = 4

GQA（分组查询注意力）：K/V 只有 4 个头，Q 有 8 个头。每 2 个 Q 头共享 1 套 K/V。节省 KV Cache 显存约 50%。

intermediate_size

FFN 中间层维度。公式 ceil(768 × π / 64) × 64 ≈ 2432。用 π 是为了让 intermediate_size 不能被 hidden_size 整除，避免某些数值对齐带来的训练捷径。

rope_theta = 1e6

RoPE 位置编码的频率基数。越大，模型对长距离位置信息越敏感。LLaMA-2 用 10000，新模型通常用 500000 或 1000000。

tie_word_embeddings = True

让 embed_tokens.weight 和 lm_head.weight 共享同一个矩阵。减少参数量：vocab_size × hidden_size = 6400 × 768 ≈ 5M 参数节省一份。

MoE 专用参数：num_experts=4、num_experts_per_tok=1 只在 use_moe=True 时生效。router_aux_loss_coef=5e-4 是负载均衡损失的权重，防止所有 token 只用同一个专家。
练习：在 Python 里跑 from model.model_minimind import MiniMindConfig; c = MiniMindConfig(); print(c.intermediate_size)，验证 intermediate_size 的值是多少。
为什么 hidden_size 要是 64 的倍数？
Tensor Core 是 NVIDIA GPU 上专门做矩阵乘法的硬件单元。它要求矩阵维度是 16/32/64 的倍数才能触发高效的 warp-level 矩阵乘法（WMMA 指令集）。

hidden_size = 768 = 64 × 12，既满足 12 个注意力头（每头 64 维），又满足 Tensor Core 对齐要求。如果 hidden_size=700（不是 64 的倍数），GPU 会用效率更低的普通矩阵乘法，速度可能降低 30-50%。
intermediate_size 为什么用 π？
intermediate_size = ceil(768 × π / 64) × 64 = 2432

用 π（无理数）是为了避免整数比例导致权重矩阵的"周期性共振"。如果用整数比（如 768 × 4 = 3072），FFN 矩阵的列空间和 hidden_size 之间存在整数对齐关系，可能造成某些特征维度被"过采样"。用 π 保证 intermediate_size/hidden_size 永远是无理数，等比缩放时始终保持非整数比。
weight_tying: lm_head.weight ≡ token_embeddings.weight → 参数节省 = vocab_size × hidden_size = 6400 × 768 = 4.9M

RMSNorm — 现代 LLM 的归一化方式

归一化层让神经网络训练更稳定。理解 RMSNorm 比理解 LayerNorm 更简单，但效果相当甚至更好。

为什么需要归一化？深层网络里，每一层的输出数值分布会随着训练漂移（有时变得极大或极小），导致梯度爆炸或消失。归一化把每层输入的数值"拉回"到合理范围，训练才能稳定。
RMSNorm(x) = x / √( mean(x²) + ε ) × γ

vs LayerNorm

LayerNorm 要算均值和方差两步（x - mean(x) 再 / std(x)），RMSNorm 只算均方根一步。计算更快，LLaMA / Mistral / GPT-4 全都改用 RMSNorm。

weight 参数 γ

初始化为全 1 的可学习向量，维度 = hidden_size。让模型自己学习"每个维度该放大多少"。这是归一化之后的"缩放还原"。

float32 中间计算

第 60 行：self.norm(x.float()) 先转 float32，算完再 .type_as(x) 转回 bfloat16。防止 bfloat16 的低精度导致 rsqrt 计算出错。

eps = 1e-5（或 1e-6）

防止分母为 0。当 x 全为 0 时，√(0 + ε) 仍有意义。Config 里用的是 rms_norm_eps = 1e-6。

输入 [B, T, 768] → RMSNorm → [B, T, 768] （shape 不变，但每个 token 向量被归一化了）
在代码里的位置：RMSNorm 出现了 4 次：MiniMindBlock 的两个归一化层（attention 前和 FFN 前），MiniMindModel 最终输出的归一化，以及 Attention 内部对 Q/K 的归一化（q_norm/k_norm）。
RMSNorm vs LayerNorm 的区别：
LayerNorm 计算两步：①均值 mean(x)，②方差 std(x)，每步都需要遍历所有维度，共 2 次内存读取。
RMSNorm 只计算均方根 rms(x) = √mean(x²)，只需 1 次遍历。假设中心化（均值≈0）已由残差连接维持，因此省去均值计算也无损效果。

在 hidden_size=768 时，RMSNorm 约快 15–20%。在 hidden_size=4096（LLaMA-7B）时优势更明显。LLaMA、Mistral、Qwen 全部采用 RMSNorm。
RMSNorm forward 核心公式model_minimind.py L56–60
56 def norm(self, x):
57 return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
59 def forward(self, x):
60 return (self.weight * self.norm(x.float())).type_as(x)
为什么先 .float() 再 rsqrt？
bfloat16 的精度只有约 3 位十进制有效数字（相当于 7 位二进制尾数）。当 x.pow(2).mean() 的值很小时，rsqrt 在 bfloat16 下会出现精度损失，甚至返回 inf。
转换为 float32（7 位十进制精度）计算 rsqrt 后，再用 .type_as(x) 转回原始 dtype，保证数值稳定性同时不影响后续的混合精度训练。
RMSNorm 出现在 4 个位置（完整列表）：
(1) 每个 Block 的 Attention 前：input_layernorm，归一化后才送入 Q/K/V 投影
(2) 每个 Block 的 FFN 前：post_attention_layernorm，归一化 Attention 残差后的 hidden state
(3) 模型最终输出前：final_norm（即 MiniMindModel 的 self.norm），在 lm_head 前做最后一次归一化
(4) Attention 内部 QK 归一化：q_norm 和 k_norm，防止 Q·K 点积数值过大，来自 Qwen2.5 的设计

RoPE 位置编码 — 让模型知道词的顺序

Transformer 的 Attention 本身是"顺序无关"的——把句子打乱顺序，Attention 看到一样的结果。位置编码解决这个问题：把位置信息注入 Q 和 K。

为什么用 RoPE？传统方法（sin/cos 绝对位置编码）把位置直接加到 embedding 上，导致 Q·K 的点积同时依赖绝对位置 m 和 n。RoPE 通过旋转让 Q_m · K_n 只依赖相对距离 (m-n)，这对长文本泛化非常关键。
q_rotated[m] = q[m] · cos(mθ) + rotate_half(q[m]) · sin(mθ)

precompute_freqs_cis

在模型初始化时运行一次，预计算所有位置的 cos 和 sin 值，存为 buffer（freqs_cos, freqs_sin）。推理时直接查表，不重复计算。

频率 θ_i = 1 / (base^(2i/dim))

每个维度对应不同的旋转频率。低维度旋转慢（编码长程位置），高维度旋转快（编码短程位置）。这和傅里叶变换的思想一样。

rotate_half

把向量的后半段移到前面并取负号，实现 90° 旋转：[-x₂, x₁]。配合 cos/sin 实现任意角度的 2D 旋转。

YaRN 长度外推

当序列长度超过训练时的 max_position_embeddings，通过线性插值部分频率实现外推。代码里 rope_scaling 参数控制这个行为，推理时可以开启。

precompute_freqs_cis(dim=96, end=32768): dim=96 → 48 个频率对 频率 = [1/θ^(0/96), 1/θ^(2/96), ..., 1/θ^(94/96)] (θ=1e6) t = [0, 1, 2, ..., 32767] (所有位置) freqs = outer(t, 频率) → [32768, 48] freqs_cos = cat([cos(freqs), cos(freqs)], dim=-1) → [32768, 96] freqs_sin = cat([sin(freqs), sin(freqs)], dim=-1) → [32768, 96] 使用时（在 MiniMindModel.forward 里）： position_embeddings = (freqs_cos[start_pos:start_pos+T], freqs_sin[start_pos:start_pos+T]) → 只取当前 batch 的位置对应的 cos/sin，shape [T, 96]
关键认知：RoPE 只作用于 Q 和 K，不作用于 V。这是因为 Q·K 的点积决定"注意力权重"（位置相关），而 V 是"值"（不需要感知位置）。
RoPE 的核心思想：让内积只依赖相对位置
传统绝对位置编码把位置向量加到 token embedding 上，导致 Q·K 的内积同时依赖位置 m 和 n，无法直接建模"距离"。

RoPE 的构造目标：找一个变换 f，使得 f(q, m) · f(k, n) = q · R_{m-n} k。
解：令 f(q, m) = R_m q，其中 R_m 是旋转矩阵。在复平面上，这等价于把每对维度 (2i, 2i+1) 旋转 m × θᵢ 弧度。两个旋转向量的内积 R_m q · R_n k = q · R_{m-n} k，只依赖相对位置差 (m-n)。
每对维度 (2i, 2i+1) 的旋转角度：θᵢ = 1 / rope_theta^(2i/d) → 低维频率快（局部模式），高维频率慢（长程依赖）
rope_theta=1,000,000 vs 传统 10,000 的区别：
θᵢ = 1 / rope_theta^(2i/d)，rope_theta 越大，θᵢ 越小，旋转越慢。
· rope_theta=10,000（原始 RoPE）：位置 0 和位置 1000 的旋转角度差已接近 2π，模型很难区分更远的位置。
· rope_theta=1,000,000（LLaMA-3 / Qwen-2 风格）：相邻 token 之间的旋转角度差很小，模型能区分数万个位置。
这是 LLaMA-3 和 Qwen-2 支持 128K 上下文长度的关键技术之一。
常见错误：把 RoPE 也应用到 V
在 Attention 计算中，只有 Q 和 K 参与位置匹配（Q·K 计算相关性），V 只负责"取出内容"（加权求和）。
V 不参与点积计算，对它做 RoPE 旋转毫无意义，还会破坏它的语义。只有 Q 和 K 需要位置编码，这是 RoPE 论文的基本设计原则。

Attention — 大模型最核心的机制

Attention 是 Transformer 的灵魂：让每个 token 决定"要借鉴序列中哪些位置的信息，借鉴多少"。理解这一层，你就理解了 70% 的 Transformer。

直觉理解：想象你在理解"苹果"这个词。它可能是水果（借鉴"吃"这个词的上下文），也可能是公司（借鉴"iPhone"的上下文）。Attention 让"苹果"这个 token 去"查询"（Q）其他所有 token 的"键"（K），找到相关的，然后把它们的"值"（V）加权求和回来。
输入： x [B=2, T=10, hidden=768] ↓ 三个线性投影（无 bias）： Q = x @ W_q → [B, T, 8×96=768] 8个查询头 K = x @ W_k → [B, T, 4×96=384] 4个键头（GQA） V = x @ W_v → [B, T, 4×96=384] 4个值头（GQA） ↓ reshape 为多头格式： Q → [B, T, 8, 96] K → [B, T, 4, 96] V → [B, T, 4, 96] ↓ Q/K 各自做 RMSNorm（QK-norm） ↓ 旋转位置编码（RoPE）作用于 Q 和 K ↓ KV Cache 拼接： xk = cat([past_k, xk], dim=1) → [B, T_total, 4, 96] ↓ GQA repeat_kv： 4头 → 8头（每头复制 n_rep=2 次） xk → [B, T_total, 8, 96] xv → [B, T_total, 8, 96] ↓ transpose(1,2) Q [B, 8, T, 96] K [B, 8, T_total, 96] V [B, 8, T_total, 96] ↓ 注意力计算： scores = Q @ K^T / √96 → [B, 8, T, T_total] causal mask: 上三角设 -inf（不能看未来） attn_w = softmax(scores) → [B, 8, T, T_total] output = attn_w @ V → [B, 8, T, 96] ↓ reshape + o_proj 输出： [B, T, 768]

GQA（分组查询注意力）L94–97

Q 头数 8，KV 头数 4，比例 n_rep = 8/4 = 2。repeat_kv() 函数把 KV 从 4 头复制成 8 头。这是 LLaMA-2 引入的优化，推理时 KV Cache 显存减半。

QK-Norm L104–105

q_norm 和 k_norm 是 Attention 内部对每个头的 Q/K 做的 RMSNorm。这是来自 Qwen2.5 等新模型的设计——防止 Q·K 点积数值过大，训练更稳定。

KV Cache L120–123

推理时，历史 token 的 K/V 不需要重新算，缓存起来。每步只送最新的 1 个 token 进来，用 cat([past_k, new_k]) 拼接。生成长文本的核心加速技术。

Flash Attention L109, L125–126

用 PyTorch 内置的 F.scaled_dot_product_attention。用 tiling 减少 GPU HBM 读写，比手写 softmax 快 2-4×。条件：seq_len > 1 且没有 KV Cache（因为 Flash 不支持 mask + past kv 的混合场景）。

Causal Mask L129

torch.full((T, T), -inf).triu(1) 生成上三角 -inf 矩阵。加到 scores 上后，softmax 这些位置的权重变 0。这保证 token i 只能看到位置 0..i 的信息，不能看到未来。

/ √head_dim L128

scores 除以 √96 ≈ 9.8。防止 Q·K 点积数值随 head_dim 增大而变得很大，导致 softmax 梯度消失（softmax 在输入极大时会趋近 one-hot，梯度接近 0）。

[B,8,T,96]@ [B,8,96,T]/ √96 [B,8,T,T]softmax [B,8,T,T]@ [B,8,T,96]→ reshape → [B,T,768]
Flash Attention 如何节省显存？
标准 Scaled Dot-Product Attention 需要在 HBM（GPU 显存）中实例化完整的 [T, T] 注意力矩阵。当 seq_len=2048、8 个头、float16 时，单层的注意力矩阵就占用 2048 × 2048 × 8 × 2 bytes ≈ 64MB。8 层合计超过 500MB，这在训练长序列时是巨大的瓶颈。

Flash Attention 利用 GPU L1/L2 缓存（SRAM）的 tiling 技术，将 Q/K/V 分成小块，在 SRAM 中分块计算 softmax 和加权求和，从不在 HBM 中保存完整的 [T,T] 矩阵。峰值显存从 O(T²) 降到 O(T)，同时由于减少了 HBM 读写次数，速度也更快（快 2-4×）。
Flash Attention 调用方式model_minimind.py L125–126
109 use_flash = (self.flash_attn and q.shape[1] > 1)
125 output = F.scaled_dot_product_attention(xq, xk, xv, attn_mask=None,
126 dropout_p=0.0, is_causal=True)
127 # is_causal=True 自动应用因果掩码，无需手动创建 triu 矩阵
GQA（分组查询注意力）的显存节省：
标准 MHA（多头注意力）：每层 KV Cache = 2 × T × num_heads × head_dim = 2 × T × 8 × 96
GQA（4个KV头共享给8个Q头）：每层 KV Cache = 2 × T × 4 × 96，节省 50%

如果 batch_size=8，总节省 = 8 × 50% = 显存缩减至原来的 1/2（相对 batch）。在推理场景下，KV Cache 通常是显存瓶颈，GQA 是支持大 batch 推理的核心技术。LLaMA-2 70B 使用 GQA 后推理显存从 140GB 降到约 70GB。
KV Cache 工作流程详解：
第 0 步（prefill）：输入完整提示词（seqlen=N），计算并保存所有 8 层的 K/V 张量，shape 为 [B, N, 4, 96]。这一步计算量最大，是 O(N²)。
第 1~T 步（decode）：每次只输入 1 个新 token，forward 只计算该 token 的 Q，然后与缓存的 K/V 做注意力（K = cat([past_K, new_k])）。每步计算量 O(N+t)，而不是 O((N+t)²)。
整体时间复杂度从 O(T² × layers) 降到 O(N² × layers + T × N × layers)，对于长文本生成加速极为显著。

FeedForward / SwiGLU — 每个 token 的"深度加工"

Attention 负责 token 之间的信息交流，FFN 负责每个 token 自己的"深度推理"。研究表明 FFN 是模型存储"知识"的地方。

为什么 FFN 能存储知识？神经科学研究发现，FFN 的行为类似于键值存储器：某些神经元在特定概念出现时激活（比如一个神经元专门在"法国"这个概念出现时激活，另一个在"首都"时激活）。
SwiGLU(x) = down_proj( SiLU(gate_proj(x)) × up_proj(x) )

三个矩阵的角色

  • gate_proj: 768 → 2432（升维，门控信号）
  • up_proj: 768 → 2432（升维，信息流）
  • down_proj: 2432 → 768（降维，投影回去）

SwiGLU vs 原版 FFN

原版 FFN（GPT-2）：ReLU(x @ W1) @ W2
SwiGLU（LLaMA）：SiLU(x @ W_gate) × (x @ W_up) 然后 @ W_down

门控机制让模型"选择性地"激活信息流，更强的表达能力。

SiLU 激活函数

SiLU(x) = x × sigmoid(x)，比 ReLU 平滑，允许小的负值通过（不完全截断），梯度更稳定。也叫 Swish。

bias=False

三个 Linear 层都不用偏置项，节省参数。现代大模型普遍做法，配合 RMSNorm 效果一样好。

[B,T,768] gate_proj →[B,T,2432] × up_proj →[B,T,2432] down_proj →[B,T,768]
SwiGLU 的门控机制：
gate_proj(x) 经过 SiLU 激活后，输出一个每个维度都在 (-∞, +∞) 的"软掩码"（SiLU 输出可正可负，正值代表"开放"，负值代表"抑制"）。
这个掩码与 up_proj(x)（信息流）逐元素相乘，动态决定每个特征维度"开放多少信息"。

与原版 ReLU FFN 相比，SwiGLU 的门控是输入依赖的（不同的 token 会产生不同的门控模式），这让 FFN 具备了类似注意力的选择性——对不同类型的输入激活不同的"知识神经元"。
SwiGLU: FFN(x) = down_proj( SiLU(gate_proj(x)) ⊗ up_proj(x) ) ← 3 个线性层，无偏置
FFN 参数占比分析：
单层 FFN 参数 = 3 × hidden_size × intermediate_size = 3 × 768 × 2432 = 5,603,328 ≈ 5.6M
8 层 FFN 总参数 ≈ 5.6M × 8 = 44.8M，约占总参数的 45%。

FFN 被认为是模型的"键值记忆"（Key-Value Memory）：gate_proj/up_proj 的每一行是一个"键"（对应某种概念/模式），down_proj 的对应列是"值"（该概念对应的输出激活）。Attention 负责"查找哪个键被激活"，FFN 负责"提取对应的值并输出"。
为什么没有 bias？
在 RMSNorm 之后，输入已经被归一化（分布中心化，方差统一），bias 是冗余的——RMSNorm 的可学习缩放参数 γ 已经能提供"偏移"的自由度。
更重要的是：量化时（INT8/INT4 推理），bias 需要额外的量化参数（scale + zero_point），会增加误差源。去掉 bias 使量化更简单、更精确。LLaMA、Mistral、Qwen 都去掉了所有 FFN 的 bias。

MOEFeedForward — 混合专家，更大的容量

MoE（Mixture of Experts）是 GPT-4、Mixtral、DeepSeek 采用的技术。核心思想：把一个大 FFN 换成 N 个小 FFN（专家），每个 token 只激活其中 Top-K 个，参数量翻 N 倍但计算量不变。

为什么 MoE 有效？不同的专家会"专注"于不同类型的 token（代码/数学/语言）。路由器学会把对应类型的 token 送给对应的专家，实现专业化。这比一个大 FFN 处理所有情况更有效率。
x [B×T, 768] ↓ gate = Linear(768, 4) → 4个专家的原始分数 ↓ softmax scores [B×T, 4] → 每个 token 对应的专家概率 ↓ topk(k=1) topk_idx [B×T, 1] 被选中的专家编号 topk_weight [B×T, 1] 被选中专家的权重 并行处理每个专家： Expert 0: 找到选了我的 token → FeedForward → 加权写回 y Expert 1: 找到选了我的 token → FeedForward → 加权写回 y Expert 2: ... Expert 3: ... aux_loss（负载均衡损失）： 鼓励 4 个专家被均匀使用，防止"专家崩塌" load = 每个专家被选中的频率 aux_loss = (load × mean_score).sum() × 4 × 5e-4

专家崩塌问题

如果没有 aux_loss，模型可能只用其中 1-2 个专家，其他专家永远不被使用（因为它们从没被训练过）。负载均衡损失强迫所有专家均匀参与。

训练时的 dummy gradient

L169–170：如果某个专家在当前 batch 完全没被选中（not mask.any()），训练时仍然加一个 0 * sum(params) 的假梯度，防止 DDP 报错（参数必须都参与反向传播）。

这里的 MoE 和 DeepSeek 的区别：这里是最简单的 Top-1 路由，没有共享专家。DeepSeek-MoE 有专门的 shared_expert 让所有 token 都经过，同时再选 K 个专家。更复杂但效果更好。
为什么需要负载均衡损失（aux_loss）？
如果没有任何约束，MoE 路由器会收敛到只使用 1-2 个专家——这叫"专家崩塌"（Expert Collapse）。原因是一个正反馈循环：
· 某个专家偶然处理了更多 token → 获得更多梯度更新 → 变得更强
· 更强的专家 → 路由器更倾向于选它 → 更多 token → 更多梯度……

其他专家由于从未被选中，参数从未被训练，永远保持初始化状态，白白占用显存。aux_loss 强制每个专家接收相近数量的 token，打破这个正反馈。
L_aux = N_experts × coef × Σᵢ(fᵢ × pᵢ) 其中 fᵢ = token分配比例，pᵢ = 平均路由概率。最小值在 fᵢ=pᵢ=1/N 时取到（完美均衡）。
虚拟梯度技巧（Straight-Through Estimator 变体）：
路由器输出 gates = softmax(linear(x))，然后用 topk(gates) 选择专家。topk 是离散操作（argmax），本身无法求导。

代码中通过 output = y + gates - gates.detach() 把梯度"附着"到离散选择上：在前向传播时等价于 y（正常输出），但反向传播时梯度会流经 gates（可导的 softmax 输出），让路由器参数能被 aux_loss 训练。这是离散优化中常见的 Straight-Through Estimator 技巧的变体。

MiniMindBlock — 把所有组件组装成一层

一个 Block 是 Transformer 的基本单元，包含 Attention + FFN，加上两个关键设计：Pre-Norm 和残差连接。

Pre-Norm（前归一化）

先做 RMSNorm，再做 Attention/FFN。原版 Transformer 是 Post-Norm（先做 Attention 再 Norm）。Pre-Norm 训练更稳定，尤其是深层网络，不容易梯度爆炸。

残差连接（Residual）

hidden_states += residual。让梯度直接从深层流回浅层，解决"梯度消失"问题。本质上是"每层只学习增量，而不是重写表示"。ResNet 发明于 2015 年，彻底改变深度学习。

input hidden_states [B, T, 768] │ ├──── residual = hidden_states (保存副本) │ ↓ input_layernorm(hidden_states) ← Pre-Norm ↓ self_attn(...) → hidden_states, present_kv ↓ hidden_states += residual ← 第一个残差加 │ ├──── 新的 residual = hidden_states (自动) │ ↓ post_attention_layernorm(hidden_states) ← Pre-Norm ↓ mlp(...) → FFN 或 MoE ↓ hidden_states = hidden_states + mlp_output ← 第二个残差加 │ output: hidden_states [B, T, 768] (shape 完全没变！)
关键认知：每个 Block 的输入和输出 shape 完全相同（都是 [B, T, hidden_size]）。这就是为什么可以把 N 个 Block 直接串联——每个 Block 只是在原有表示上叠加细微修正。
Pre-Norm 为什么比 Post-Norm 更稳定？
Post-Norm（GPT-2 风格）：x = Norm(x + Attn(x))。梯度在反向传播时必须先通过 Norm 层，Norm 的缩放参数 γ 初始化为 1，但训练中可能变小甚至趋近 0。当 γ 很小时，梯度被大幅缩放，深层的梯度传回浅层时几乎消失。

Pre-Norm（LLaMA 风格）：x = x + Attn(Norm(x))。残差流 x 有一条直接通路，梯度可以不经过任何 Norm 层直接从第 L 层反向传播到第 0 层。无论网络多深，浅层都能收到有效梯度信号。
更深的模型（>12 层）几乎都用 Pre-Norm，这是现代大模型能训练上百层的关键。
残差连接的物理意义：
x = x + Attn(Norm(x)) 这个公式意味着：网络每层只需要学习"差量" Δ = Attn(Norm(x))，而不是完整的变换。
· 如果 Attn 输出全 0，x 原样通过（恒等映射），这是最容易学习的情形——这解释了为什么残差网络更容易优化
· 第 0 层的信息可以不衰减地传递到任意深度（通过残差流）
· 梯度可以从任意深度直接回传到第 0 层，不会消失
8 层 Block 的分工（经验规律）：
· 前 2 层（Block 0-1）：学习局部语法和词法特征——识别词性、基本词组、标点边界
· 中间 4 层（Block 2-5）：学习语义组合——理解短语含义、实体关系、句子结构
· 最后 2 层（Block 6-7）：学习任务特定的预测模式——根据上下文决定生成什么词

可以通过实验验证：冻结前 2 层训练（保留局部特征学习），或冻结后 2 层（保留任务特定知识），观察不同冻结策略对模型性能的影响。

MiniMindModel — 完整的 Transformer 主干

把所有组件串联起来：Token → Embedding → N 层 Block → 最终 RMSNorm → hidden_states。

输入： input_ids [B, T] (整数，值在 [0, vocab_size) 范围内) ↓ embed_tokens: Embedding(6400, 768) 每个 token id 查表，得到 768 维向量 → hidden_states [B, T, 768] ↓ dropout (训练时随机 mask，推理时跳过) ↓ 预计算 RoPE： position_embeddings = (freqs_cos[start_pos:start_pos+T], freqs_sin[start_pos:start_pos+T]) ↓ ┌──────────────────────────────────────────────────────┐ │ Block 0: Attention + FFN │ │ Block 1: Attention + FFN │ │ ... │ │ Block 7: Attention + FFN │ │ 每层收集 present_kv，组成 presents 列表 │ └──────────────────────────────────────────────────────┘ ↓ norm: 最终 RMSNorm ↓ 返回: hidden_states [B, T, 768], presents (KV Cache), aux_loss

start_pos（KV Cache 位置）

推理时，start_pos = past_kvs[0][0].shape[1]，即已缓存的 token 数量。用这个值从 freqs_cos 里取正确位置的 RoPE 旋转角，确保位置编码连续。

register_buffer

self.register_buffer("freqs_cos", ...) 把预计算的 cos/sin 注册为模型的"缓冲区"——不是参数（不更新梯度），但会随 model.to(device) 自动移动到 GPU，并在 state_dict 中序列化。

aux_loss 累积

遍历所有 MoE 层，把每层的 mlp.aux_loss 加起来。这个 loss 会被传到 ForCausalLM，再加到总 loss 里参与反向传播。

RoPE buffer 重计算

L216–218 的特殊处理：用 transformers≥5.x 的 meta device 初始化时，buffer 可能丢失（全 0）。检测到后重新计算。

register_buffer vs nn.Parameter 的区别：
nn.Parameter：会出现在 optimizer.param_groups 中，参与梯度计算和参数更新，在 model.parameters() 中可见。
register_buffer：不参与梯度更新，不出现在 model.parameters() 中，但会随 model.to(device) 自动移动到正确的 device，并在 state_dict 中序列化保存。

RoPE 的 cos/sin 缓存是固定的数学表（由 rope_theta 和序列长度唯一确定），不需要学习，用 register_buffer 可以让它们：①随模型一起 save_pretrained / from_pretrained，②推理时自动在正确 device 上，③不占用优化器状态的显存。
start_pos 的作用：
在 KV Cache 推理模式下，每次只输入 1 个新 token，但 RoPE 需要知道这个 token 在序列中的绝对位置。
start_pos = past_kvs[0][0].shape[1]（已缓存的 token 数量），然后取 freqs_cos[start_pos : start_pos+T]。
这保证位置编码的连续性：prefill 阶段 token 0~N-1 获得正确的旋转角，decode 阶段每个新 token 获得正确的位置 N, N+1, N+2, ...
DDP 与 checkpoint 加载顺序陷阱：
必须先 load_state_dict，再用 DDP 包装模型。
原因：DDP（DistributedDataParallel）会给所有参数名加上 module. 前缀（如 attn.qkv 变成 module.attn.qkv）。
如果先 DDP 再加载，原来保存的 state_dict（没有 module. 前缀）会与模型的参数名不匹配，strict=True 时直接报错，strict=False 时所有参数都加载失败，模型相当于随机初始化。
正确顺序：model = MiniMindForCausalLM(config) → model.load_state_dict(...) → model = DDP(model)

MiniMindForCausalLM — 语言模型头 + 训练损失

在 MiniMindModel 之上加一个线性层（lm_head），把 hidden_states 映射到词表概率，计算下一个 token 的预测损失。

语言模型的学习目标：给定 token 序列 [t₁, t₂, ..., tₙ]，模型预测每个位置的下一个 token。损失 = cross_entropy(logit[t], target[t+1])。这叫"下一个 token 预测"，是现代 LLM 通用的预训练目标。

lm_head = Linear(768, 6400, bias=False)

把每个位置的 768 维 hidden state 线性投影到 6400 维（词表大小）。输出叫 logits，softmax 后是每个词的概率。注意没有 bias。

权重共享（tie_word_embeddings）

L242：self.model.embed_tokens.weight = self.lm_head.weight。Embedding 矩阵和 lm_head 矩阵共享同一份参数。直觉：一个 token 的输入向量和它的预测向量应该在同一个语义空间里。节省 6400×768 ≈ 5M 参数。

交叉熵损失 L250–252

x = logits[..., :-1, :]（去掉最后一个预测）
y = labels[..., 1:]（去掉第一个标签）
这是"右移对齐"：位置 i 的 logit 对应位置 i+1 的标签（预测下一个）。

ignore_index = -100

被 mask 掉的位置（prompt 部分、padding）的 label 设为 -100，交叉熵自动忽略这些位置，不计算 loss。这是 SFT 的核心技术。

logits_to_keep 优化：L247–248，只对最后 N 个 token 计算 logits。推理时只需要最后 1 个 token 的 logits，可以节省大量计算。
ignore_index=-100 的数学原理：
PyTorch 的 cross_entropy(logits, labels, ignore_index=-100) 在计算时，对 label=-100 的位置完全跳过：分子（被预测 token 的 log 概率）和分母（损失的归一化分母）都不包含这些位置。

这使得提示词（Prompt）部分的 token 不参与梯度计算——模型只从"回答"部分学习如何生成。这是 SFT（监督微调）的核心技术：把 prompt 对应的 label 设为 -100，让模型专注于学习从 prompt 推导出正确 response 的能力，而不是死记 prompt 内容。
Loss = -1/|答案tokens| × Σ_{t ∈ 答案} log P(yₜ | y₁,...,yₜ₋₁) ← 只在 label≠-100 的位置求平均
logits_to_keep 的计算节省：
lm_head 是一个 [hidden_size, vocab_size] = [768, 6400] 的矩阵乘法。如果 seq_len=2048，完整计算需要 2048 × 768 × 6400 次浮点运算。
推理时只需要最后 1 个 token 的 logits（用于采样下一个 token），设置 logits_to_keep=1 后只做 1 × 768 × 6400 的矩阵乘法，节省 2047/2048 ≈ 99.95% 的 lm_head 计算量。在长上下文推理中收益极大。
权重绑定（weight tying）的梯度流：
lm_head.weight 和 embed_tokens.weight 共享同一个内存地址（不是复制，是同一个张量）。
反向传播时，来自 lm_head 的梯度（"预测错了哪个词"）和来自 embedding 查找的梯度（"哪个词被输入"）叠加在同一个矩阵上，使得词嵌入空间和输出预测空间保持一致——一个词的"输入表示"和"输出预测向量"应该编码相同的语义，这个约束通过共享参数被天然满足。

generate() — 自回归生成的完整实现

这是推理的核心：用训练好的模型，逐个 token 地生成文本。理解采样策略，你才能控制模型的输出风格。

generate() 主循环： 初始: input_ids = prompt tokens [B, prompt_len] finished = [False, False, ...] loop (最多 max_new_tokens 次): 1. forward(input_ids[:, past_len:], past_key_values) → 利用 KV Cache，只送新 token → logits [B, 1, vocab_size] 2. logits = logits[:, -1, :] / temperature → 取最后一个位置 → [B, vocab_size] → 温度缩放 3. repetition_penalty：已出现词降分 4. top_k：只保留最高 k 个词，其余 -inf 5. top_p（nucleus sampling）： 累计概率 > p 的词截掉 6. next_token = multinomial(softmax(logits)) → 从概率分布采样 1 个 token 7. input_ids = cat([input_ids, next_token]) 8. 如果 next_token == eos: finished[i] = True 9. 如果全部 finished: break

temperature（温度）

logits / temperature。
temperature → 0：越来越确定性（总选最大概率）
temperature = 1：按原分布采样
temperature > 1：更随机多样（趋向均匀分布）

默认 0.85，略有随机性。

top_k 采样

保留概率最高的 k=50 个词，其余设为 -∞。
防止模型选到极低概率的怪词，同时保留多样性。
如果 k=1 等同于 greedy decoding。

top_p（核采样）

对所有词按概率降序排，累加到 p=0.85 就截断。比 top_k 更自适应：分布集中时（模型很确定）保留少数几个词，分布平坦时保留更多。

repetition_penalty

对已出现过的 token，分数 > 0 时除以 rp，分数 < 0 时乘以 rp。rp > 1 降低重复概率，rp = 1.0 无效果。防止模型一直重复说同一句话。

@torch.inference_mode()

推理时不需要计算梯度。这个装饰器比 no_grad() 更激进——完全禁止梯度追踪，节省约 20% 内存，速度更快。

KV Cache 的使用

L264–265：past_len = past_kvs[0][0].shape[1]，每步只送 input_ids[:, past_len:]（新 token）而不是完整序列。随着生成，每步的 forward 计算量是 O(1) 而不是 O(T²)。

练习：修改 temperature 为 0.1（贪婪）和 2.0（极随机），对比同一个 prompt 的输出。这直观展示了采样参数的作用。
温度采样的数学原理：
logits / T 等价于在 softmax 之前缩放每个 logit。设最高 logit 为 a，第二高为 b（a > b）：
· T → 0：(a-b)/T → ∞，softmax 变成 argmax，始终选最大概率（贪心解码）
· T = 1：原始分布，按模型训练时学到的概率采样
· T → ∞：所有 logit 除以大数后趋近相同，softmax 趋向均匀分布（完全随机）

实践中：创意写作 T=0.8–1.2，代码/数学 T=0–0.3（需要精确性），对话 T=0.5–0.8（流畅但确定）。
Nucleus sampling（top_p）：找最小集合 V* 使得 Σ_{v∈V*} P(v) ≥ p，从 V* 中采样。top_p=0.9 意味着"每次只从累计概率 90% 的候选词中选"，自动适应词汇量的宽窄。
重复惩罚（repetition_penalty）公式：
对于已经在生成序列中出现过的 token t：
· 如果 logit[t] > 0：logit[t] /= rp（降低正分数，降低概率）
· 如果 logit[t] < 0：logit[t] *= rp（使负分数更负，进一步降低概率）

rp=1.0 无效果；rp=1.3 明显减少重复（适合大多数场景）；rp>1.5 可能强制模型不断引入新词，导致偏离主题甚至语言不连贯。建议从 rp=1.1 开始调试。
常见错误：生成前忘记 model.eval()
训练模式（model.train()）下：
· Dropout 仍然随机激活，每次推理结果不可重现，且质量下降
· BatchNorm（如果有）使用 batch 统计而非运行统计，batch_size=1 时行为异常
· 梯度追踪仍然开启，消耗额外显存

正确的推理前准备：model.eval()（关闭 dropout 和 batchnorm 的训练行为）+ torch.no_grad() 或 @torch.inference_mode()（关闭梯度追踪）。inference_mode 比 no_grad 更激进，完全禁用自动求导引擎，推理速度约快 5–10%。
